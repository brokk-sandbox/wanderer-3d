# User Notes

Use this folder for **human-authored notes** (design thoughts, testing observations, playtest feedback, roadmap ideas).

## Guidelines
- Keep these notes distinct from agent/tooling notes in `docs/devlog/`.
- Use clear file names like `playtest-notes.md`, `design-notes.md`, or date-based names.
- Keep entries concise and task-oriented.

## Suggested format
- Date
- What you changed
- Why
- Result / observations
- Next step

This keeps long-term design discussion separate from technical change logs.
